###1. Create a Pod named nginx-pod using image nginx:1.25. 
   Constraints
    Namespace: dev
    Container name: web
    Port: 80
    Label: app=frontend

###Ans: 
apiVersion: v1
kind: Namespace
metadata:
  name: dev
---
apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod
  namespace: dev
  labels:
    app: frontend
spec:
  containers:
  - name: web
    image: nginx:1.25
    ports:
    - containerPort: 80

kubectl create namespace dev
kubectl run nginx-pod \
  --image=nginx:1.25 \
  --namespace=dev \
  --port=80 \
  --labels=app=frontend


###2. Create a ReplicaSet named rs-nginx with 3 replicas.
   Constraints
     Image: nginx:1.25
     Label: app=web
     Container name: nginx-container

###Ans: 
apiVersion: apps/v1
kind: ReplicaSet
metadata:
  name: rs-nginx
spec:
  replicas: 3
  selector:
    matchLabels:
      app: web
  template:
    metadata:
      labels:
        app: web
    spec:
      containers:
      - name: nginx-container
        image: nginx:1.25

Imperative Not recommended / not supported properly

###3. Scale ReplicaSet to 5 replicas (imperative only)

Ans:
kubectl scale rs rs-nginx --replicas=5

Verification
kubectl get rs rs-nginx
kubectl get pods -l app=web


###4. Update image to nginx:1.26 (imperative)
Ans:
kubectl edit rs rs-nginx


###5. Create multi-container Pod

6: Create a Deployment named nginx-deploy.
Constraints
  Image: nginx:1.25
  Replicas: 3
  Label: app=nginx
  Container name: nginx-container 

###Ans:
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deploy
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx-container
        image: nginx:1.25

kubectl create deployment nginx-deploy --image=nginx:1.25
kubectl scale deployment nginx-deploy --replicas=3 
kubectl label deployment nginx-deploy --labels=app=nginx 


7: Update the Deployment nginx-deploy image to nginx:1.26.
Constraints
  Use imperative command only
  Do NOT edit YAML manually
  Ensure zero downtime

Ans: kubectl set image deployment nginx-deploy nginx-container=nginx:1.26


8: Expose the Deployment nginx-deploy as a ClusterIP Service.
Constraints
  Service name: nginx-svc
  Port: 80
  TargetPort: 80
  Selector must match Deployment
Ans: 
  apiVersion: v1 
  kind: Service
  metadata: 
    name: nginx-svc
  spec: 
    type: clusterIP
    selector: 
          app: nginx 
    ports:
    - targetPort: 80
      port: 80

kubectl expose deployment nginx-deploy \
  --name=nginx-svc \
  --port=80 \
  --target-port=80 \
  --type=ClusterIP

9: Scale Deployment nginx-deploy to 5 replicas.

Constraints
  Use imperative command
  Verify rollout status

Ans: kubectl scale deployment nginx-deploy --replicas=5
     kubectl rollout status deployment nginx-deploy 

10: Configure rolling update strategy.

Constraints
  maxUnavailable: 1
  maxSurge: 2
  Apply using YAML (not imperative)

Ans: 
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deploy
spec:
  replicas: 5
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxUnavailable: 1
      maxSurge: 2
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx-container
        image: nginx:1.26
