###1. Create a Pod named nginx-pod using image nginx:1.25. 
   Constraints
    Namespace: dev
    Container name: web
    Port: 80
    Label: app=frontend

###Ans: 
apiVersion: v1
kind: Namespace
metadata:
  name: dev
---
apiVersion: v1
kind: Pod
metadata:
  name: nginx-pod
  namespace: dev
  labels:
    app: frontend
spec:
  containers:
  - name: web
    image: nginx:1.25
    ports:
    - containerPort: 80

kubectl create namespace dev
kubectl run nginx-pod \
  --image=nginx:1.25 \
  --namespace=dev \
  --port=80 \
  --labels=app=frontend


###2. Create a ReplicaSet named rs-nginx with 3 replicas.
   Constraints
     Image: nginx:1.25
     Label: app=web
     Container name: nginx-container

###Ans: 
apiVersion: apps/v1
kind: ReplicaSet
metadata:
  name: rs-nginx
spec:
  replicas: 3
  selector:
    matchLabels:
      app: web
  template:
    metadata:
      labels:
        app: web
    spec:
      containers:
      - name: nginx-container
        image: nginx:1.25

Imperative Not recommended / not supported properly

Verification
kubectl get rs rs-nginx
kubectl get pods -l app=web


###4. Update image to nginx:1.26 (imperative)

10: Configure rolling update strategy.

Constraints
  maxUnavailable: 1
  maxSurge: 2
  Apply using YAML (not imperative)

  name: nginx-deploy

  replicas: 5

    type: RollingUpdate

21. Create a Pod named affinity-pod.

Constraints
  Image: nginx
  Schedule only on nodes with label: disktype=ssd
  Use nodeAffinity (requiredDuringSchedulingIgnoredDuringExecution)

Ans: 
apiVersion: v1 
kind: Pod 
metadata: 
   name: affinity-pod
spec: 
   affinity:
      nodeAffinity:
	    requiredDuringSchedulingIgnoredDuringExecution: 
		   nodeSelectorTerms: 
		    - matchExpressions: 
			     key: disktype 
				 operator: In 
				 values: 
				    - ssd 
   containers:
     -  name: nginx 
	    image: nginx 

kubectl label nodes node_name disktype=ssd
